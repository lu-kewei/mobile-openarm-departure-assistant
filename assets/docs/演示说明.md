# 演示说明

## 1. 演示目标

演示系统用于展示家庭出门物品准备助手的系统逻辑，而不是展示真实机器人控制。答辩时重点说明：用户输入清单后，系统如何结合环境与习惯数据做建议，如何生成机器人作业流程，以及如何处理状态反馈和异常。

## 2. 演示入口

打开 [demo/index.html](../../demo/index.html)。页面为静态网页，不需要后端服务，不连接真实 API。

## 3. 演示场景

- 雨天上班：输入钥匙、钱包、耳机，天气选择下雨，系统推荐雨伞。
- 会议出门：场景选择会议，系统推荐电脑、文件夹、证件。
- 老人就医：场景选择就医，系统推荐药盒、眼镜、证件。
- 普通外出：展示基础清单解析和机器人准备流程。

## 4. 操作步骤

1. 在物品清单输入框中输入“钥匙、钱包、耳机”。
2. 选择出门场景，例如“上班”。
3. 设置天气、交通和日程模拟信息。
4. 点击“生成任务”，查看系统推荐和最终清单。
5. 点击“开始模拟执行”，观察任务状态、执行步骤和日志。
6. 打开页面下方的架构展示区，说明系统、硬件和软件架构关系。

## 5. 页面区域说明

- 清单与场景区：模拟用户输入和出门背景。
- 环境信息区：模拟天气、交通和日程输入。
- 推荐区：展示系统建议及理由。
- 任务流程区：展示机器人从解析到放置的作业步骤。
- 状态面板：显示待处理、执行中、已完成和异常数量。
- 日志区：按时间线记录系统决策和执行过程。
- 架构展示区：链接系统流程、硬件架构、软件架构和家庭场景图。

## 6. 模拟数据说明

数据位于 [data](../../data) 目录：

- `mock_items.json`：常见出门物品、标签、位置和抓取方式。
- `mock_environment.json`：天气、交通、日程和习惯数据。
- `mock_tasks.json`：预设答辩演示任务。

## 7. 答辩讲解顺序

建议按“场景痛点 - 系统边界 - 总体架构 - 多模态识别 - 作业流程 - 安全异常 - 演示页面 - 可扩展性”顺序讲解。讲解时要强调本项目是系统设计与模拟演示，不宣称已经实现真实 ROS、机械臂控制或视觉算法。
