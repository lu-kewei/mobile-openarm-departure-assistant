# 系统设计手册

## 1. 系统边界

本项目是“系统设计 + 非实物演示”项目，不连接真实 Mobile OpenARM，不执行真实 ROS 控制、机械臂控制、视觉识别或室内导航代码。导航、识别、抓取、天气、交通和日程均通过模块设计、流程图和模拟数据展示。

系统的真实交付内容包括：文档、Mermaid 图、SVG 展示图、模拟 JSON 数据和静态网页演示。系统的模拟行为用于说明模块关系、输入输出、状态流转和异常处理，不代表已经完成真实机器人部署。

## 2. 角色与使用场景

| 角色 | 需求 | 系统响应 |
| --- | --- | --- |
| 上班用户 | 准备钥匙、钱包、手机、耳机 | 解析清单，规划搜索顺序，集中放置到玄关 |
| 雨天出门用户 | 需要避免忘带雨具 | 根据天气建议加入雨伞，等待用户确认 |
| 会议用户 | 需要携带办公材料 | 根据日程建议电脑、文件夹、证件 |
| 老年用户 | 需要药品、眼镜、手机 | 根据场景和习惯生成提醒 |
| 赶时间用户 | 需要出行时间判断 | 根据交通拥堵提醒提前出门 |

## 3. 硬件架构

Mobile OpenARM 在本方案中作为移动操作平台，由以下硬件模块组成：

| 硬件模块 | 设计职责 | 说明 |
| --- | --- | --- |
| 移动底盘 | 室内移动、路径跟踪、避障 | 模拟导航到玄关、客厅、卧室、书房等区域 |
| 机械臂 | 抓取、搬运、放置 | 仅做流程设计，不实现真实运动控制 |
| 夹爪 | 适配小物体、薄片物品、长条物品 | 钥匙、钱包、雨伞、文件夹使用不同抓取策略 |
| RGB-D 相机 | 视觉识别、深度定位、三维坐标获取 | 在设计中输出物品类别、置信度和空间位置 |
| 电子标签读取模块 | RFID / BLE / NFC / UWB 身份确认 | 作为视觉补充，解决遮挡和相似物品混淆 |
| 计算单元 | 运行任务规划、感知融合、决策和交互模块 | 负责汇总各模块状态 |
| 语音或前端设备 | 接收清单、展示状态、确认建议 | 演示系统用网页替代 |
| 安全传感器 | 碰撞检测、近距离障碍感知、急停 | 家庭环境下安全优先 |

电子标签不是视觉识别的替代品。视觉用于判断“这是什么物品”，标签用于确认“是不是目标物品”，习惯模型用于预测“它最可能在哪里”。

## 5. 机器人作业流程

1. 用户输入物品清单。
2. 系统解析清单，生成初始任务。
3. 读取模拟天气、交通、日程和用户习惯。
4. 自主决策模块判断是否推荐额外物品。
5. 用户确认最终清单。
6. 系统根据家庭语义地图和常见位置生成搜索顺序。
7. 机器人导航到第一个目标区域。
8. 先用电子标签判断目标物品是否可能在附近。
9. 再用视觉识别和 RGB-D 深度定位确定物品位置。
10. 目标置信度足够时进入抓取规划。
11. 目标不确定时切换搜索区域或向用户询问。
12. 机械臂根据物品类型选择抓取策略。
13. 抓取后移动到玄关整理区。
14. 放置物品并进行视觉确认。
15. 更新任务状态，继续下一个物品。
16. 全部完成后生成最终提醒和出门建议。

### 异常流程

| 异常 | 判断条件 | 处理策略 |
| --- | --- | --- |
| 找不到物品 | 常见区域均搜索失败 | 切换备用区域，并提示用户可能缺失 |
| 视觉置信度低 | 类别置信度低于阈值 | 靠近、调整视角、结合标签再次判断 |
| 标签有信号但视觉未识别 | 标签匹配，视觉无目标 | 靠近遮挡区域，降低移动速度，等待用户确认 |
| 抓取失败 | 夹爪闭合异常或物品滑落 | 最多重试一次，随后标记异常 |
| 路径被阻挡 | 避障模块持续无法通行 | 等待、重规划或提示用户清理通道 |
| 用户拒绝推荐 | 用户取消建议物品 | 不加入最终清单，保留日志 |

## 6. 多模态物品识别机制

```text
视觉识别：判断“这是什么物品”
电子标签：确认“是不是目标物品”
习惯模型：预测“它最可能在哪里”
```

融合规则：

- 视觉置信度高且标签匹配：直接确认目标。
- 视觉置信度高但标签不匹配：提示可能是同类物品，不抓取。
- 标签匹配但视觉未识别：机器人靠近或调整视角后再判断。
- 二者都不确定：按习惯位置切换搜索区域。
- 易碎或高价值物品：即使识别成功，也需要用户确认后执行抓取。

## 7. 自主决策机制

自主决策采用“建议 - 确认 - 执行”模式。系统不替用户擅自决定，只给出可解释建议。

| 规则 | 建议 |
| --- | --- |
| 天气为下雨 | 推荐雨伞 |
| 天气降温 | 推荐外套 |
| 交通拥堵 | 提醒提前出门 |
| 日程包含会议 | 推荐电脑、文件夹、证件 |
| 日程包含就医 | 推荐药盒、证件、病历 |
| 工作日习惯携带某物 | 推荐加入清单 |

## 8. 数据设计

物品数据字段：

```json
{
  "id": "keys",
  "name": "钥匙",
  "category": "随身物品",
  "hasTag": true,
  "tagType": "RFID",
  "tagId": "RFID-KEY-001",
  "commonLocations": ["玄关柜", "客厅茶几"],
  "graspType": "小物体夹取",
  "priority": 1
}
```

环境数据包含天气、温度、降雨概率、交通状态、预计延迟和日程类型。任务数据包含用户输入、场景、最终清单、任务状态和当前步骤。用户习惯数据包含出门类型、常带物品、常见位置和推荐权重。

## 9. 安全机制

- 移动速度限制，靠近人和家具时降低速度。
- 避障和急停优先级高于任务执行。
- 机械臂工作空间限制在安全范围内。
- 抓取失败不强行执行，最多进行有限重试。
- 易碎、高价值或不确定目标需要用户确认。
- 人或宠物靠近时暂停机械臂动作。
- 识别结果不确定时不抓取。
- 用户隐私数据采用本地存储和最小化使用原则。

## 10. 测试与验证

| 测试用例 | 输入 | 预期输出 | 验证重点 |
| --- | --- | --- | --- |
| 清单解析 | “钥匙、钱包、耳机” | 三个结构化物品 | 中文分隔、去重、标准化 |
| 雨天推荐 | 天气为下雨 | 推荐雨伞 | 推荐理由清晰，可拒绝 |
| 会议推荐 | 日程为会议 | 推荐电脑、文件夹、证件 | 日程规则正确 |
| 搜索路径 | 最终清单含钥匙、钱包 | 玄关、客厅优先 | 常见位置排序 |
| 多模态识别 | 视觉高置信 + 标签匹配 | 确认目标 | 融合逻辑正确 |
| 标签异常 | 标签匹配但视觉失败 | 靠近或询问 | 不盲目抓取 |
| 抓取失败 | 模拟抓取失败 | 标记异常并记录日志 | 异常可解释 |
| 完整演示 | 雨天上班任务 | 生成最终清单和流程日志 | 闭环展示 |
